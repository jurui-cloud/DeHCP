   # FG-SSPF 完整算法设计与实现蓝图

## 一、 整体数据流动 (Data Flow Pipeline)

整个协同感知系统在单个 Ego 智能体 $i$ 的视角下，数据的前向传播遵循以下生命周期：

1. **异构感知与编码 (Heterogeneous Encoding)：**

   场景中的每个智能体 $j$ 利用其自带的传感器（$m_j$）获取原始数据 $X_j$，并通过对应的编码器提取 BEV 特征：$F_j = E_{m_j}(X_j)$。

2. **空间对齐 (Spatial Alignment)：**

   邻居智能体的特征通过 V2X 传输给 Ego 智能体，并利用位姿矩阵 $\Gamma_{j \to i}$ 转换到 Ego 坐标系：$\tilde F_{j \to i} = \Gamma_{j \to i}(F_j)$。

3. **FG-SSPF 解耦特征提取：**

   对齐后的特征送入 FG-SSPF 模块，分别流经 **Shared（共享）** 和 **Specific（特有）** 两个平行的分支，提取出共性特征 $H_i^{sh}$ 和特性补充特征 $H_i^{sp}$。

4. **门控融合与检测 (Gate Aggregation & Detection)：**

   利用门控网络 $\alpha_i$ 将两者融合得到 $F_i^{out}$，最终送入 Detection Head 输出 3D 边界框。

------

## 二、 核心模块设计与网络层参数 (Module Design)

假设输入对齐特征 $\tilde F_{j \to i}$ 的维度为 $B \times C_{in} \times H \times W$。

### 1. 模态嵌入模块 (Modality Embedding)

- **功能：** 将离散的模态 ID（如 0代表64线LiDAR，1代表相机）转化为连续的稠密向量。
- **网络层：** `nn.Embedding(num_modalities, emb_dim)`
- **参数维度：** 假设 `emb_dim` = 16。输入 $m_j$，输出 $e_j \in \mathbb{R}^{16}$。
- **空间广播：** 经过一层 `nn.Linear(16, C_{in})` 将维度映射到特征图通道数，并进行 `unsqueeze` 扩展，最终得到形状为 $B \times C_{in} \times 1 \times 1$ 的张量 $\bar e_j$。

### 2. 前景估计器 (Foreground Estimator)

- **功能：** 提取关键目标区域，滤除背景噪声。
- **网络层：** 一个轻量级的卷积网络。
  - `Conv2d(C_{in}, C_{in}//2, kernel_size=3, padding=1)` + `ReLU`
  - `Conv2d(C_{in}//2, 1, kernel_size=1)` + `Sigmoid`
- **输出维度：** 前景掩码 $M_{fg} \in [0, 1]^{B \times 1 \times H \times W}$。

### 3. Shared Branch (共享协议分支)

- **共享投影层 (Shared Projector)：**
  - `Conv2d(C_{in}, C_{hidden}, kernel_size=1, bias=False)` + `BatchNorm` + `ReLU`
  - **特征过滤：** $Z^{sh} = \text{Projector}(\tilde F_{j \to i}) \otimes M_{fg}$ （逐元素相乘）。
- **共享金字塔 (Shared Pyramid Fusion)：**
  - 多尺度 ResNeXt 结构。接收所有智能体过滤后的 $Z^{sh}$，在多尺度下进行特征相加（或注意力融合），输出统一空间特征 $H_i^{sh} \in \mathbb{R}^{B \times C_{out} \times H_{out} \times W_{out}}$。

### 4. Specific Branch (特有协议分支)

- **模态条件化 (Modality Injection)：** $F^{cond} = \tilde F_{j \to i} \oplus \bar e_j$ （逐元素相加）。
- **特有投影层 (Specific Projector)：**
  - 结构同 Shared Projector：`Conv2d(C_{in}, C_{hidden}, kernel_size=1)`。
  - **特征过滤：** $Z^{sp} = \text{Projector}(F^{cond}) \otimes M_{fg}$。
- **特有金字塔 (Specific Pyramid Fusion)：**
  - 与 Shared Pyramid **结构严格对称**。输出特有补充特征 $H_i^{sp} \in \mathbb{R}^{B \times C_{out} \times H_{out} \times W_{out}}$。

### 5. 门控聚合模块 (Gate Aggregation)

- **功能：** 用全局共性引导特性特征的自适应吸收。
- **数据流与网络层：**
  1. 对 $H_i^{sh}$ 进行全局平均池化 `AdaptiveAvgPool2d(1)` $\rightarrow B \times C_{out}$。
  2. 提取 Ego 智能体的模态向量 $e_i \in \mathbb{R}^{16}$。
  3. 拼接两者得到门控输入，维度为 $B \times (C_{out} + 16)$。
  4. 通过 `MLP`：`Linear(..., hidden)` + `ReLU` + `Linear(hidden, C_{out})`。
  5. 通过 `Sigmoid` 激活，得到 $\alpha_i \in (0, 1)^{B \times C_{out} \times 1 \times 1}$。
- **最终融合：** $F_i^{out} = H_i^{sh} \oplus (\alpha_i \otimes H_i^{sp})$。

## 三、 损失函数设计 (Loss Functions)

两阶段训练采用**相同**的总损失函数公式体系，但参与计算和更新的模块不同。不给 Specific 支路加独立的检测约束。

$$L_{total} = L_{det}^{fused} + \lambda_{fg} L_{fg} + \lambda_{aux} L_{det}^{shared} + \lambda_{gate} L_{gate}$$

1. **最终融合检测损失 ($L_{det}^{fused}$):**
   - 作用于网络终点。用 $F_i^{out}$ 经过主检测头，计算与 3D Ground Truth 的 Focal Loss (分类) 和 Smooth-L1 Loss (回归)。
2. **前景掩码损失 ($L_{fg}$):**
   - 将 3D 边界框投射到 BEV 空间生成 0/1 的二值 Mask ($M_{GT}$)。
   - 计算预测的 $M_{fg}$ 与 $M_{GT}$ 之间的 Binary Cross Entropy (BCE) 损失。
3. **共享分支辅助监督 ($L_{det}^{shared}$) - 核心创新:**
   - 作用于 $H_i^{sh}$。它需要经过一个**辅助检测头 (Auxiliary Head)** 输出预测结果，同样计算 Focal Loss 和 Smooth-L1。
   - 目的：强制 Shared 空间具备完全独立的检测能力。
4. **门控稀疏正则化 ($L_{gate}$):**
   - 计算 $\alpha_i$ 在通道维度的 L1 范数平均值：$L_{gate} = \frac{1}{C_{out}} \sum |\alpha_i|$。
   - 目的：压制噪声，鼓励非必要不开启门控。

## 四、 两阶段训练参数冻结控制 (Training Dynamics)

这是代码实现中最重要的一步，需要通过 `requires_grad` 严格控制。

### Stage 1: 协作基础训练 (Collaboration Base Training)

- **目标：** 建立统一基座和门控规则。
- **网络状态：** 所有的 Encoder、FG-SSPF 模块、Detection Heads 均为 **可训练状态 (`requires_grad = True`)**。

### Stage 2: 异构新智能体接入 (Backward Alignment)

- **目标：** 将新模态以极低成本对齐到统一空间。
- **冻结模块 (Frozen `requires_grad = False`)：**
  - `Foreground Estimator`
  - `Shared Projector` 与 `Shared Pyramid`
  - `Gate Aggregation`
  - 主从两个 `Detection Head`
  - `nn.Embedding` 中**已存在的旧模态**索引（例如 $e_0$）。
- **可训练模块 (Trainable `requires_grad = True`)：**
  - 新接入智能体的传感器 `Encoder`。
  - `nn.Embedding` 中为新模态**新开辟**的索引（$e_{new}$）。
  - `Specific Projector` (作为新模态身份的轻量级适配器解冻)。

## 五、 代码实现的“黑魔法”与避坑指南

在编写 PyTorch 代码时，请务必注意以下两点：

1. **Gate MLP 的偏置初始化：**

   为了防止门控在训练初期剧烈震荡，必须将 `Gate Aggregation` 最后一个全连接层的 Bias 初始化为负数。

   Python

   ```
   nn.init.zeros_(self.gate_mlp[-1].weight)
   nn.init.constant_(self.gate_mlp[-1].bias, -2.0)
   # 这会使训练初期 alpha 约为 0.11，网络优先训练 Shared 分支。
   ```

2. **Auxiliary Head 的推理屏蔽：**

   $L_{det}^{shared}$ 对应的辅助检测头只存在于 `model.train()` 模式中。在写 `forward` 函数时，加入判断：

   Python

   ```
   if self.training:
       aux_preds = self.aux_head(H_sh)
       # 计算辅助 loss...
   ```

   推理测试阶段 (`model.eval()`) 直接跳过该计算，以实现零延迟增加。
